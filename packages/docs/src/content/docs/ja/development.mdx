`gh-vibe` は pnpm モノレポです。CLI は `packages/cli/`、ドキュメントサイト
（このサイト）は `packages/docs/` に置かれています。

## セットアップ

前提: flakes を有効にした [Nix](https://nixos.org/download/)。
[Determinate Systems インストーラ](https://github.com/DeterminateSystems/nix-installer)
は既定で有効化します。既存環境では `~/.config/nix/nix.conf` に
`experimental-features = nix-command flakes` を追記してください。
[direnv](https://direnv.net/) は任意です — 一度 `direnv allow` すれば
`nix develop` を打たずとも `cd` 時に自動でシェルが読み込まれます。

```sh
nix develop             # flake.nix から node, pnpm, bun を導入（または `direnv allow`）
pnpm install            # ワークスペースの依存をインストール
```

## CLI を実行

```sh
pnpm -C packages/cli dev review 123 --dry-run
```

## テスト + 型チェック

```sh
pnpm -C packages/cli test    # bun test
pnpm -C packages/cli check   # tsc --noEmit
```

## バイナリをビルド

```sh
pnpm -C packages/cli build   # リポジトリルートに ./gh-vibe を生成
./gh-vibe --version
```

## ドキュメントを編集

```sh
pnpm -C packages/docs dev     # http://localhost:4321
pnpm -C packages/docs build   # 静的成果物 → packages/docs/dist
pnpm -C packages/docs check   # astro check
```

## リリース

`v*` タグを push すると `release` ワークフローが `darwin-{arm64,amd64}`,
`linux-{arm64,amd64}`, `windows-amd64` 向けにクロスコンパイルし、対応する
GitHub Release にバイナリをアップロードします。`gh extension install` は
`<name>-<os>-<arch>[.exe]` の命名規則からプラットフォームに合うアセットを
選びます。
