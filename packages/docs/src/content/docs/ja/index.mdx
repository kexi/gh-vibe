## 特長

<CardGrid stagger>
  <Card title="PR → worktree" icon="git-pull-request">
    `gh vibe review 123` で PR の head ブランチ（fork PR も対応）を fetch
    し、`vibe start --reuse` で worktree を生成します。
  </Card>
  <Card title="Issue → worktree" icon="bookmark">
    `gh vibe issue 42` で issue のラベルとタイトルから
    `<type>/<num>-<slug>` のブランチ名を導出し、worktree を作成します。
  </Card>
  <Card title="merge 済み worktree を一括掃除" icon="trash">
    `gh vibe clean` は PR が merge / close 済みの vibe worktree を見つけて
    削除します。件数入力の確認プロンプトつきで、誤削除を防ぎます。
  </Card>
  <Card title="シェル自動 cd" icon="terminal">
    bash / zsh / fish / PowerShell 用スニペットを仕込めば、`gh vibe review`
    成功時に自動で作成済みの worktree に `cd` します。
  </Card>
  <Card title="既定で安全" icon="approve-check">
    ラッパーはバージョン付きセンチネルで囲まれた出力のみを `eval` するため、
    無関係な `gh` 標準出力がシェルコードとして実行されることはありません。
  </Card>
  <Card title="マルチプラットフォーム配布" icon="rocket">
    darwin / linux / windows × arm64 / amd64 のビルド済みバイナリを
    `gh extension install` で導入できます。
  </Card>
</CardGrid>

## クイックスタート

```sh
gh extension install kexi/gh-vibe

# 任意: 以下の行を ~/.zshrc（または ~/.bashrc）に追記しておくと、
# `gh vibe review` / `gh vibe issue` 成功時にシェルが worktree に自動で
# cd します:
eval "$(gh vibe shell-setup)"

gh vibe review 123
```

> `gh`、`git`、[`vibe`](https://github.com/kexi/vibe) が `PATH` に必要です。
